Simple plot of the convergence of given values. Two plotting modes depending on the arguments given:

Logger: - Takes min, max, mean, and standard deviations of objectives from the BasicLogger object and plots them. Both: - Provided value will override

  • std_objective (Union[ndarray, Series, None]) – Vector of objective values standard deviations.
  • logger (Optional[BasicLogger]) – BasicLogger, the above objectives will be taken from here.
  • ax (Optional[Axes]) – Axes, optional axes.

optimizer.plotting.trajectory module¶

Best solution at each iteration plots.

optimizer.plotting.trajectory.best_trajectory_plot(best_trajectory_logger, penalties, bounds, normalize_penalties=False, variable_labels=None)[source]¶

Generates a plot on how the optimization was executed, containing 4 charts:

  • The first chart is the objective value as function of the iteration in logger
  • The second captures how far from its bounds were the optimizable varibles in a heatmap. The darker the shade, the closer the value to the upper bound.
  • The third captures the distance of each constraint to it’s bound (inequality penalty). The darker the shade, the closer the constraint was to the bound –– in other words, the tighter the constraint. If the bound is violated, then the occurrence is colored in red.
  • The fourth captures the impact of the penalties on the objective value. The darker the shade, the more this penalty contributed to the objective value. If the penalty was not triggered (value = 0), then the occurrence is colored in blue.
Parameters:
  • best_trajectory_logger (BestTrajectoryLogger) – The logger that logged during the optimization execution.
  • penalties (List[Penalty]) – A list of Penalty objects used for slack and penalty plots.
  • bounds (List[Tuple]) – A list of (min_bound, max_bound) for each of the variables. This has to be in the same order as the solutions columns.
  • normalize_penalties (bool) – If True, this will make a normalization for each Penalty. This might be useful if at some iteration one of the penalties happens to be very large and thus the shades in the heatmap towards the end of the run get very light. To read it better, it could be interesting to normalize these values.
  • variable_labels (Optional[List[str]]) – labels list for adding them on the y axis of the variable vs. their bounds plot.
Return type:

(<class ‘matplotlib.figure.Figure’>, typing.List[matplotlib.axes._axes.Axes])
Returns:

A tuple (plt.Figure, List[plt.Axes]) with the plots.
