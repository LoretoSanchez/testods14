class optimizer.stoppers.no_improvement.NoImprovementStopper(patience, sense, min_delta=0.0, first_delta=0.0)[source]¶

Bases: optimizer.stoppers.base.BaseStopper

This class will stop the search after N iterations without improvement in the best objective value.

__init__(patience, sense, min_delta=0.0, first_delta=0.0)[source]¶

Constructor.

Parameters:
  • patience (int) – number of iterations without improvement.
  • sense (str) – “minimize” or “maximize”.
  • min_delta (Real) –

    minimum difference to be considered an improvement.

    • Zero means any improvement will be reset the patience counter.
    • Must be positive, any improvement will be compared in the appropriate direction whether we are maximizing or minimizing.
  • first_delta (Real) –

    minimum difference between the first best solution to be considered an improvement.

    • Counting up toward patience will not begin until a solution that is at least first_delta better than the first best solution.
Raises:
  • ValueError – if sense is not “minimize” or “maximize”.
  • ValueError – if min_delta is negative.
  • ValueError – if first_delta is negative.

optimizer.stoppers.satisfied_constraint module¶

Stopper which terminates when population meets constraints.

class optimizer.stoppers.satisfied_constraint.SatisfiedConstraintsStopper(sense, top_n=1)[source]¶

Bases: optimizer.stoppers.base.BaseStopper

This class will stop the search if all constraints are met. It allows the optimization to stop when the top_n members of the parameter population satisfy all the constraints.

Parameters:
  • top_n (int) – Number of best solutions to check for constraint violations
  • sense (str) – Whether to “minimize” or “maximize” the objective.

Examples >> stopper = SatisfiedConstraintsStopper(sense=’maximize’) >> while not solver.stop() or stopper.stop(): >> parameters = solver.ask() >> objective_values = problem(parameters) >> solver.tell(parameters, objective_values) >> stopper.update(solver, problem)

optimizer.stoppers.utils.top_n_indices(x, sense, top_n)[source]¶

Utility for find the indices of the top_n elements of a vector, when that vector is sorted according to sense.

Argsort sorts in ascending order. For maximize, we take the last top_n elements. For minimization, we sort then take the first top_n elements. :type x: Union[ndarray, Series] :param x: Vector of values to sort and find best idx for. :type sense: str :param sense: Whether to maximize or minimize the function :type top_n: int :param top_n: How many top solutions to check for constraint violations
