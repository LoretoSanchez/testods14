optimizer.utils.diagnostics.evaluate_on_historic_data(data, problem, bounds)[source]¶

Evaluates the problem statement on historical data. This is helpful to sanity check the constraints and problem set-up. :type data: DataFrame :param data: input data :type problem: OptimizationProblem :param problem: Optimization Problem :type bounds: List[Tuple] :param bounds: List of bounds for each variables in the input data. It is expected :param that the bounds are ordered tuples: :type that the bounds are ordered tuples: i.e. (min, max :param each column in the input data.):

Return type:

DataFrame
Returns:

A dataframe with input data along with the evaluated penalties, slack, bounds check and objective values. If the inequality constraint is violated, the slack is NaN.
Raises:
  • ValueError – if number of columns in data does not match
  • the number of bounds

Safely assigns a column of a matrix to the provided values.

  • values (Union[ndarray, Series]) – values to assign.
  • col (Union[str, int]) – which column to put values into.
  • row_mask (Union[ndarray, Series, List[~T], None]) – optional row mask. Must be a boolean type with len(row_mask) == parameter.shape[1].

Union[ndarray, matrix, DataFrame]

Matrix, with the updated column.
Raises:
  • ValueError – if values is 1 dimensional.

optimizer.utils.initializer.latin_hypercube(shape, rng=None)[source]¶

Uses latin hypercube sampling from [0, 1)^d. See here for more: https://en.wikipedia.org/wiki/Latin_hypercube_sampling

Code taken from: scipy.optimize._differentialevolution.DifferentialEvolutionSolver.

Parameters:
  • shape (Tuple[int]) – shape of the problem, (n_samples, n_dimensions).
  • rng (Union[int, None, RandomState]) –

    supports the same typing as sklearn.utils.check_random_state.

    • int: seed for a new RandomState.
    • np.RandomState, will be used as the random generator.
    • None: the current RandomState will be used.
Return type:

ndarray
Returns:

np.ndarray with the specified shape.
