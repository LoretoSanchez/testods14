Plotting¶

The plots built in to Optimizer are intended to help users better understand their algorithms’ performance. Use the plots described here to help tune solver hyperparameters or debug issues with objectives and constraints.

Convergence plot¶

The convergence_plot visualizes summary statistics of objective values over the iterations of the optimization. This plot can be used to tune hyperparameters in a search or simply confirm that a search has converged after a certain number of iterations.

[1]:

import matplotlib.pyplot as plt
from optimizer.loggers import BasicLogger
from optimizer.plotting import convergence_plot
from optimizer.testing.problems import Rastrigin
from optimizer.solvers import ParticleSwarmSolver

For demonstration, we’ll minimize the Rastrigin function with ParticleSwarmSolver.

[2]:

seed = 0
dimension = 5
f = Rastrigin()
bounds = f.bounds(dimension)

logger = BasicLogger()
pso = ParticleSwarmSolver(bounds, maxiter=100, seed=seed)

while not pso.stop():
    x = pso.ask()
    f_x = f(x)
    pso.tell(x, f_x)
    logger.log(pso)

We can then pass the BasicLogger object to convergence plot. The convergence_plot works similar to other plotting libraries where an optional matplotlib Axis object can be provided. In the case it is not provided, the current global axis is used.

[3]:

_, ax = plt.subplots(figsize=(10, 7))
convergence_plot(logger=logger, ax=ax);

The values plotted above are aggregated summary statistics of the solver’s current population at each iteration. More specifically: - Minimum Objective: “overall” minimum objective value. Calculated as logger.log_df["min_objective"].expanding().min(). In other words, the all time minimum value found at the current iteration. - Maximum Objective: “overall” maximum objective value. This is the same calculation as the minimum objective value. - Mean Objective: mean objective value of the current population and the standard deviation as a cloud around the mean value.

If one of these values is not exactly what you want, it can easily be replaced.

[4]:

_, ax = plt.subplots(figsize=(10, 7))
convergence_plot(logger=logger, overall_max_objective=logger.log_df["max_objective"], ax=ax);

This now shows the maximum objective of each population, rather than the overall maximum of the search.

Penalty plot¶

The penalty_plot visualizes the value of the calculated penalties over the iterations of the search. This plot is helpful for debugging things like overly restrictive constraints, constraints that have no effect, and if a certain constraint is not satisfied at the end of the search.

[5]:

from optimizer import penalty
from optimizer.plotting import penalty_plot
from optimizer.loggers import PenaltyLogger
from optimizer.utils.functional import column

We’ll put in a few penalties so we have something to log during the optimization.

[6]:

penalties = [
    penalty(
        lambda x: x[:, 0] ** 2 + x[:, 1] ** 2, "<=", 10,
        name="$x_1^2 + x_2^2 \leq 10$"
    ),
    penalty(
        lambda x: x[:, 2] + x[:, 3], ">=", -10,
        name="$x_3 + x_4 \geq -10$"
    )
]

[7]:

logger = PenaltyLogger()
pso = ParticleSwarmSolver(bounds, maxiter=100, seed=seed)

while not pso.stop():
    x = pso.ask()
    f_x = f(x) + sum(p(x) for p in penalties)
    pso.tell(x, f_x)
    logger.log(penalties)

Then we can use PenaltyLogger object and penalty_plot to visualize the calculated penalty values over time. The convergence_plot works similar to other plotting libraries where an optional matplotlib Axis object can be provided. In the case it is not provided, the current global axis is used.

[8]:

_, ax = plt.subplots(figsize=(10, 7))
penalty_plot(logger, normalize_penalties=False, ax=ax);

This plot shows us the average penalty value in each iteration. From this plot we can see that the penalty for \(x_1^2 + x_2^2 \leq 10\) does not quite reach zero. Therefore, the solution we found after 100 generations might still violate this constraint. Conversely, we can see that the penalty for \(x_3 + x_4 \geq -10\) never seems to be a nonzero value. Practically speaking, we could consider removing this constraint if appropriate.

The penalty_plot also allows us to plot normalized values. This simply does min-max scaling to set all values to be in \([0, 1]\).

[9]:

_, ax = plt.subplots(figsize=(10, 7))
penalty_plot(logger, normalize_penalties=True, ax=ax);

From this normalized view, we can see that the penalty for \(x_3 + x_4 \geq -10\) was actually violated a few times, and remained violated at the end of the search.
