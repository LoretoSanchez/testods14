Constraints¬∂

Consider the optimization problem:

\[\begin{split}\begin{align*} &\min & f(\mathbf{x}) \\ &\text{s.t.} & g(\mathbf{x}) \leq c \\ && x_i \in S \end{align*}\end{split}\]

for some set \(S\). Optimizer‚Äôs syntax makes a clear distinction between a constraint and how it is handled. You‚Äôre encouraged to write out your optimization problem in a form that looks like the one written above. However, in code we offer two methods for handling constraints:

  • penalty: adds a constraint-dependent penalty term to the problem‚Äôs objective function.
  • repair: converts an infeasible solution to a feasible one through some transformation.

This notebook will give a high level overview of both. See the penalty user guide and the repair user guide for more details.

Constraints as penalty functions¬∂

If we take the same optimization problem above, we can reformulate the constraint \(g(\mathbf{x}) \leq c\) as a penalty term in the objective:

\[\begin{split}\begin{align*} &\min & f(\mathbf{x}) + \lambda P(\mathbf{x}) \\ & \text{s.t.} & x_i \in S \end{align*}\end{split}\]

Where \(P(\mathbf{x})\) can take many forms and \(\lambda > 0\). An example of a linear penalty is \(P(\mathbf{x}) = \max(0, g(\mathbf{x}) - c)\).

Let‚Äôs say our \(g(\mathbf{x}) = \sum_i x_i\) and \(c = 10\). We can formulate this as a penalty in Optimizer using the penalty function.

[1]:

import numpy as np
import pandas as pd
from optimizer import penalty

def row_sum(x_):
    return np.sum(x_, axis=1)

[2]:

sum_penalty = penalty(row_sum, "<=", 10)

[3]:

type(sum_penalty)

[3]:

optimizer.constraint.penalty.Penalty

The penalty function returns a callable Penalty object. We can then apply sum_penalty to a matrix of decision variables:

[4]:

matrix = np.array([
    [1, 1, 1],
    [5, 5, 5],
    [10, 10, 10]
])
sum_penalty(matrix)

[4]:

array([ 0.,  5., 20.])

[5]:

sum_penalty.constraint_func(matrix)

[5]:

array([ 3, 15, 30])

The penalty is of course only applied when the constraint is violated, so the first row has a penalty of \(0\). The other two rows are penalized based on their distance from the constraint boundary.

See the user guide for more information on penalty.

Constraints as repair functions¬∂

Generally speaking, penalties are for situations when the function applied to the decision variables of an optimization problem are complicated. For example, a penalty function would be useful if one‚Äôs constraint function is a machine learning model.

For set membership constraints like \(x_i \in S\), it is clear how to modify \(\mathbf{x}\) to ensure it satisfies the given constraint. We can simply set \(x_i\) to its nearest neighbor in the set \(S\).

Let‚Äôs use a more concrete example where \(S = \mathbb{Z}\), the set of all integers and \(i = 1\).

[6]:

from optimizer import repair
from optimizer.utils.functional import column
from optimizer.constraint.sets import Integers

[7]:

int_repair = repair(column(0), "in", Integers())

Optimizer provides the column function for indexing Numpy arrays. When using Pandas a DataFrame, simply specify the column:

[8]:

int_repair_pd = repair("column_0", "in", Integers())

In both cases, repair returns a callable Repair object which performs the expected operation. In this case, both objects round the first column.

[9]:

data = np.random.randn(5, 3)

Warning about pickling constraints¬∂

If you‚Äôre going to be working with Python‚Äôs multiprocessing, note that all of your constraints will need to be pickleable. This (among other things) means you cannot pickle constraint that uses a lambda function in the constraint definition ‚Äì see the Python docs for more on objects that can be pickled. There are three options to get around this:

  1. Use fully qualified functions defined at the top level of a module.
  2. If you‚Äôre just indexing one column, consider using the column utility function in optimizer.utils.functional or simply provide a string to constraint for DataFrame objects.
  3. Switch to a multiprocessing solution that can handle lambdas like Dask.distributed.
