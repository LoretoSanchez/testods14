Warm start¬∂

Perhaps you have a starting point for your problem that you know is a good solution and want to use that information. Warm start involves providing this prior information to the search with the intention of guiding it toward better solutions.

Problem setup¬∂

We‚Äôll use ParticleSwarmSolver and the Max test function to demonstrate one approach to warm starting a search. First, we‚Äôll set up our problem.

[1]:

import numpy as np

from optimizer.testing.problems import Max
from optimizer.solvers import ParticleSwarmSolver
from optimizer.utils.initializer import latin_hypercube

seed = 0
dimension = 5

[2]:

f = Max()
bounds = f.bounds(dimension)
bounds

[2]:

[(0, 10000), (0, 10000), (0, 10000), (0, 10000), (0, 10000)]

Warm starting ParticleSwarmSolver¬∂

Since the domain for this problem is pretty large, we‚Äôll help the search out by giving it a good solution to the problem.

[3]:

warm_start = np.array([25 for _ in range(dimension)])

Then we‚Äôll give the solver our solution in before starting the usual ask and tell loop.

[7]:

solver = ParticleSwarmSolver(bounds, seed=0)

initial_parameters = solver.ask()

initial_parameters[0] = warm_start
initial_objectives = f(initial_parameters)

solver.tell(initial_parameters, initial_objectives)

[8]:

min(initial_objectives)

[8]:

25.0

Our provided solution was the best row in the initial parameters. Now the solver knows about our good solution and we can continue the search as normal.

[10]:

while not solver.stop():
    parameters = solver.ask()
    objective_values = f(parameters)
    solver.tell(parameters, objective_values)

solver.best()
